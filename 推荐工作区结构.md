# 推荐工作区结构

如果你准备认真用这个 Skill，建议先把本地项目文件夹放整齐。

不是为了好看，是为了少返工。

## 一个顺手的放法

```text
我的书项目/
├─ 00-项目说明/
│  └─ 本书说明.md
├─ 01-目录/
│  └─ 正书目录.docx
├─ 02-写作规范/
│  ├─ 写作格式规范.doc
│  └─ 正文模板.doc
├─ 03-参考资料/
│  ├─ 参考书1.pdf
│  ├─ 参考书2.pdf
│  ├─ 样章.docx
│  └─ 术语表.md
├─ 04-章节草稿/
│  ├─ 第1章.docx
│  ├─ 第2章.docx
│  └─ 第3章.docx
└─ 05-终稿输出/
```

## 每个文件夹放什么

### `01-目录`

放全书目录。

最好只有一个最权威版本，不要同时放 3 份“目录最终版”“目录最终版2”“目录真的最终版”。

### `02-写作规范`

放出版社或你自己定下来的格式要求。

比如：
- 标题层级
- 字数要求
- `本章要点` 怎么写
- 段落长度偏好
- 图表要求

### `03-参考资料`

放真正要参考的东西，不要把无关资料也堆进去。

建议放：
- 事实来源
- 同类书样章
- 术语表
- 你希望模仿的行文风格样本

### `04-章节草稿`

每章单独一个 Word 文件最省心。

这样 Skill 改稿时不容易串文件，也方便你逐章审。

### `05-终稿输出`

如果你想把“草稿”和“终稿”分开，这个文件夹会很好用。

## 为什么建议这么放

因为这个 Skill 工作时，最怕两种情况：

1. 找不到哪个文件才是准的。
2. 明明有规范，但混在一堆资料里，结果谁都没先看。

文件夹一旦清楚，流程就会顺很多。

## 最推荐的调用方式

你可以这样说：

```text
使用 $book-manuscript-writer。
先读取“01-目录”里的目录文件、
再读取“02-写作规范”里的格式规范和模板、
然后读取“03-参考资料”里的材料，
最后撰写第2章，并写入“04-章节草稿”中的 Word 文档。
```

如果你只想审校，不想重写：

```text
使用 $book-manuscript-writer。
只对“04-章节草稿”里的第2章做终审校，
重点检查措辞、逻辑、段落长度和 AI 味，
改完后写回原文件；如果原文件被占用，就另存新文件。
```

## 一句话建议

资料别乱放，文件别重名，目录别反复改名。

你整理清楚一次，后面会省很多命。
