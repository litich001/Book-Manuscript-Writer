# 贡献说明

如果你准备继续完善这个仓库，建议按下面的方式维护：

## 优先改什么

优先修改这些内容：
- `book-manuscript-writer/SKILL.md`：主流程与行为规则。
- `book-manuscript-writer/references/checklist.md`：终检标准。
- `README.md`：面对 GitHub 访客的说明。

## 每次修改后至少检查三件事

1. 规则有没有和 README 说的不一致。
2. 字数口径、标题层级、终审校标准有没有写清楚。
3. 如果你新增了一条硬规则，是否也同步写进了 checklist。

## 推荐修改原则

- 少加空话，多加可执行规则。
- 少写“大而全”，多写“什么时候必须这么做”。
- 任何容易误判的地方，都应该显式写出来。

## 不建议做的事

- 不要把 Skill 写成宣传页。
- 不要把 README 写成术语堆砌文档。
- 不要只改一处规则，忘了同步其它文件。
